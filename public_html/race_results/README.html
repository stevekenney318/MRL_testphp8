MRL Race Results System

Backfill + Monitor + Snapshot storage for ESPN race results (MRL /race_results/ folder)

Doc Version: v1.00.00.01 Generated (America/New_York): loading… PHP Target: 7.3
Naming: R## / E## / Z## Index: <year>/_year_index.json Scripts: engine / backfill / monitor

What this system does

  • Backfill pulls an entire ESPN season year page and creates year/race folders.
  • Monitor (cron) periodically checks ESPN for the latest race and saves snapshots only when needed.
  • Snapshots are saved only on:
    • first FINAL detection (non-zero scoring)
    • hash changes (e.g., penalties / result updates)

Normal behavior

When there is no new race or no results change, cron runs quietly: logs/heartbeat update, but no new snapshots and no email.

Email behavior

Email sends only when the monitor sees non-zero scoring for the race (and again only if the hash changes later).

Folder structure

/public_html/race_results/
  race_results_engine.php
  race_results_backfill.php
  race_results_monitor.php
  _race_results_monitor.log
  _race_results_monitor_state.json
  _race_results_monitor_heartbeat.txt
  (optional) other logs / state files...

  2026/
    _year_index.json
    R01_..._<raceId>/
      _meta.json
      final_table_hash.txt
      snapshot_YYYYMMDD_HHMMSSsss.html
      snapshot_summary_YYYYMMDD_HHMMSSsss.txt
    E01_..._<raceId>/   (exhibition)
    Z01_..._<raceId>/   (anomaly / placeholder)

Each race folder contains a _meta.json and (when FINAL has been detected) snapshot files plus the last known final_table_hash.txt.

Naming rules (R / E / Z)

R## — Points races

Points races are numbered R01 through R36 (season schedule alignment). The numbering is driven by include-phrase logic and preserved by _year_index.json.

E## — Exhibitions

Exhibitions are numbered E01, E02, etc. These do not consume R numbers and are still fetched/snapshotted for long-term usefulness.

Z## — Anomalies

Rare “placeholder / duplicate listing” oddities (example: a duplicate Daytona entry with zero scoring). Z items do not consume R or E numbering.

_year_index.json

Backfill writes the index. Monitor uses it to keep naming stable and avoid renumbering mistakes. If you delete year folders, the index will be rebuilt on the next backfill.

How to run

Backfill (browser)

https://manliusracingleague.com/race_results/race_results_backfill.php?year=2026

Use this for initial population (or after a “clean slate” wipe).

Backfill (CLI)

php race_results_backfill.php 2026

Equivalent to the browser run, but via command line.

Monitor (CLI test)

php race_results_monitor.php 2026

Good for quick testing (no cron needed).

Monitor (cron)

*/10 * * * * /usr/bin/php /home/u809830586/domains/manliusracingleague.com/public_html/race_results/race_results_monitor.php 2026

Runs every 10 minutes (adjust as desired).

Best practice for “go live”

Backfill the year first (build folders + index), then enable cron monitor.

What to check when something looks wrong

No email / no new snapshots

This is normal if ESPN is still showing zero scoring or if the results haven’t changed. Check _race_results_monitor_heartbeat.txt and _race_results_monitor.log timestamps to confirm cron is running.

HTTP 500

Check _race_results_monitor_php_errors.log (or backfill equivalent). Also verify the PHP file contents didn’t get a stray character before <?php.

Unexpected extra race entry

ESPN sometimes lists odd “placeholder” items. The system uses Z## for anomalies to avoid breaking the R01–R36 schedule alignment.
