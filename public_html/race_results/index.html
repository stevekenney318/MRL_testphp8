What this system does

This folder contains a small set of PHP scripts that pull ESPN race results, store snapshots on meaningful changes, and (optionally) send an email only when “FINAL” is detected by non-zero scoring values.

Design goal: no noise. Snapshots only happen on first FINAL (per race) and on later results changes (hash change).

Files in this system

File Purpose Typical run
race_results_engine.php Shared helper functions (fetch, parse year page, final detector, file helpers) Included by monitor/backfill
race_results_backfill.php One-time / occasional: populate a full year, create folders, save snapshots when FINAL Manual (browser) or CLI
race_results_monitor.php Cron: checks latest race and only snapshots/emails when needed Automated via cron

Quick start

1) One-time: populate a year

https://manliusracingleague.com/race_results/race_results_backfill.php?year=2026

Backfill writes folders + state for that year and creates snapshots only when FINAL is detected.

2) Turn on cron (monitor)

Run monitor on a schedule (ex: every 10 minutes). The monitor should not create duplicate snapshots if nothing changes.

/usr/bin/php /home/u809830586/domains/domains/manliusracingleague.com/public_html/race_results/race_results_monitor.php

Folder naming rules (R / E / Z)

This system uses a predictable folder naming scheme to keep the year aligned to the schedule.

Prefixes

  • R## — points races (schedule-aligned numbering, e.g., R03…)
  • E## — exhibition races (stored like points races, but separate numbering)
  • Z## — anomalies (rare). Used when something looks like a duplicate/placeholder and should NOT consume R##/E##.

What triggers a snapshot?

Snapshots are stored only when they matter.

Rules

  • First FINAL for that race → snapshot saved
  • Hash change later (penalty/update) → snapshot saved again
  • If ESPN changes nothing meaningful, monitor should log activity but not create duplicates

How “FINAL” is detected

  • Find a scoring-style table containing PTS/POINTS + BONUS or PENALTY
  • Require at least one non-zero value in those columns

Helpful “sanity checks”

If you see only _meta.json and no snapshot files, it usually means: not final yet.
If you see HTTP 500, check the script’s configured PHP error log file in the same folder.

Operational notes

Recommended workflow

  • Use backfill to populate past seasons (2017–2026) quickly.
  • Use monitor via cron for ongoing season updates.
  • If you ever change folder naming rules or file outputs, you may choose to wipe year folders and re-backfill.
